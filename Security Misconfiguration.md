# Неправильная конфигурация (Security Misconfiguration)

## Цель работы
Выявить ошибки конфигурации веб-сервера и приложения, которые могут привести к раскрытию служебной информации или созданию дополнительных векторов атак.

## Ход выполнения

### 1) Разведка
Для анализа конфигурации веб-сервера был перехвачен HTTP-ответ от DVWA при обычном запросе к главной странице.

<img width="871" height="901" alt="image" src="https://github.com/user-attachments/assets/86ea96d9-28ce-4909-b766-4f59f36d9be8" />

| Заголовок | Значение | Проблема |
|-----------|----------|----------|
| Server | Apache/2.4.25 (Debian) | Раскрыта точная версия веб-сервера |
| X-Frame-Options | отсутствует | Возможность встраивания страницы в iframe |
| Content-Security-Policy | отсутствует | Нет защиты от XSS и встраивания ресурсов |
| Strict-Transport-Security | отсутствует | Нет принудительного HTTPS |

Сервер раскрывает версию программного обеспечения: `Apache/2.4.25 (Debian)`. Это также позволяет искать известные уязвимости для данной версии.

### 2) Атака
Отсутствие заголовка `X-Frame-Options` делает приложение уязвимым к атаке встраивания страницы (Clickjacking). Для демонстрации был создан HTML-файл, который загружает страницу управления уровнем безопасности DVWA (`security.php`) внутри прозрачного iframe. Поверх iframe размещена кнопка-приманка.

### 3) Эксплуатация
Созданная страница была открыта в браузере, где пользователь уже авторизован в DVWA. Пользователь видит интерфейс DVWA, поверх которого расположена кнопка «Забрать бонус». При нажатии на эту кнопку пользователь фактически взаимодействует с элементами DVWA, расположенными под курсором.

Поскольку сессия пользователя активна, все действия выполняются от его имени. Можно расположить кнопку над любым критическим элементом интерфейса — например, над кнопкой изменения уровня безопасности.

## Выводы о защищенности
В ходе работы установлено, что веб-приложение уязвимо к атаке типа Clickjacking. Причиной является отсутствие защитных HTTP-заголовков. Отсутствие механизмов позволяет встраивать страницы приложения во внешние ресурсы и выполнять действия от имени пользователя без его ведома.

Данная уязвимость классифицируется как:
- **CWE-1021: Improper Restriction of Rendered UI Layers or Frames** — отсутствие ограничений на встраивание страниц в iframe
- **CWE-200: Exposure of Sensitive Information** — раскрытие версии сервера
