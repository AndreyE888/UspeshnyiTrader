# Бесконтрольная загрузка файлов (Unrestricted File Upload)

## Цель работы
Выявить возможность загрузки и выполнения вредоносных файлов на сервере, проанализировать механизмы проверки загружаемых файлов и оценить риск компрометации сервера через неконтролируемую загрузку.

### 3) Эксплуатация

**Список загруженных файлов:**
![Скриншот 5: Директория с загруженными файлами](https://github.com/user-attachments/assets/6558b1e7-99a2-4332-ad24-019ea2d10757)

В списке файлов присутствует `1.php`, что подтверждает его успешную загрузку на сервер.

При обращении к файлу с параметром `?cmd=whoami` сервер должен выполнить PHP-код, отобразив информацию о конфигурации системы. В данном тестовом окружении выполнение PHP в директории `uploads` отключено, однако сам факт загрузки PHP-файла без проверки является критической уязвимостью.

## Выводы о защищенности
В результате анализа выявлено отсутствие механизмов проверки загружаемых файлов: не контролируется тип файла, не проверяется его содержимое, файлы загружаются в доступную директорию. Это позволяет злоумышленнику загрузить вредоносный скрипт (например, PHP-шелл) на сервер.

Данная атака классифицируется как:
- **CWE-434: Unrestricted Upload of File with Dangerous Type** — неограниченная загрузка файлов опасного типа
- **CWE-94: Improper Control of Generation of Code ('Code Injection')** — возможность выполнения загруженного кода на сервере (при соответствующей конфигурации веб-сервера)
