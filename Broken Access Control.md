# Нарушение контроля доступа (Broken Access Control)

## Цель работы
Определить возможность получения несанкционированного доступа к данным или функциям системы путём изменения параметров запроса или прямого обращения к защищённым ресурсам. Продемонстрировать эксплуатацию уязвимости.

## Ход выполнения

### 1) Разведка
Открыл свою торговую платформу UspeshnyiTrader (локальный сервер).

Выполнил регистрацию двух пользователей:
- Пользователь 1: логин: `trader1`, пароль: `123456`, почта: `trader1@gmail.com`
- Пользователь 2: логин: `trader2`, пароль: `1234567`, почта: `trader2@gmail.com`

Был обнаружен уязвимый эндпоинт: **`/Account/Profile/{id}`**

Выявил:
- идентификатор пользователя передаётся напрямую в URL;
- сервер не проверяет права доступа при запросе профиля по ID.

### 2) Атака
В ходе анализа был выявлен запрос:

`GET /Account/Profile/3` — возвращает данные профиля текущего пользователя (trader1)

В адресной строке браузера идентификатор был изменён на:

`GET /Account/Profile/1`

Сервер ответил данными пользователя с ID=8 (администратор):

```json
{
  "id": 8,
  "username": "demofixed8",
  "email": "demofixed8@example.com",
  "balance": 10000,
  "createdAt": "2026-03-13T11:45:20.572099Z",
  "isAdmin": false,
  "password": "admin123"
}
```

HTTP-статус: 200 OK

Демонстрация: trader1 (обычный пользователь) получает данные администратора:

Выводы о защищенности
В ходе тестирования веб-приложения UspeshnyiTrader была выявлена уязвимость класса Broken Access Control, соответствующая категории A01:2021 проекта OWASP Top 10. Приложение не выполняет проверку принадлежности объекта текущему пользователю при обращении к ресурсам вида /Account/Profile/{id}.

Отсутствие серверной проверки приводит к:

горизонтальной эскалации привилегий — доступ к данным других пользователей

вертикальной эскалации привилегий — получение данных администратора

несанкционированному изменению данных — модификация баланса произвольного пользователя

Данный тип уязвимости относится к категории CWE-284: Improper Access Control (неправильное ограничение прав доступа), так как система не обеспечивает надлежащую проверку прав при доступе к объектам и выполнении привилегированных операций.

Эксплуатация уязвимости не требует специальных технических навыков и может быть автоматизирована посредством перебора идентификаторов.
