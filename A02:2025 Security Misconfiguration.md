# Неправильная конфигурация (Security Misconfiguration)

## Цель работы
Выявить ошибки конфигурации веб-сервера и приложения, которые могут привести к раскрытию служебной информации или созданию дополнительных векторов атак.

## Ход выполнения

### 1) Разведка
Для анализа конфигурации веб-сервера был выбран модуль File Upload в DVWA. При попытке загрузить файлы была обнаружена особенность конфигурации сервера.

**Directory listing в директории `/hackable/uploads/`**


**Выявлено:**
- Directory listing не отключён — любой пользователь может просмотреть список всех загруженных файлов
- В директории видны файлы: `1.php`, `1.txt`, `dvwa_email.png`
- Злоумышленник может скачать любой файл для дальнейшего анализа

### 2) Атака
Согласно классификации OWASP Top 10:2025 (A05: Security Misconfiguration), к ошибкам конфигурации относится Scenario #2:

> *"Directory listing is not disabled on the server. An attacker discovers they can simply list directories. The attacker finds and downloads the compiled Java classes, which they decompile and reverse engineer to view the code. The attacker then finds a severe access control flaw in the application."*

В данном случае директория `/hackable/uploads/` доступна для просмотра. Злоумышленник может:

1. Просмотреть список всех загруженных файлов
2. Скачать файлы, содержащие конфиденциальную информацию
3. Проанализировать загруженные файлы на наличие уязвимостей

### 3) Эксплуатация
Используя открытый directory listing, злоумышленник может:

- Определить, какие типы файлов принимает сервер (в том числе PHP)
- Скачать загруженные ранее конфигурационные файлы (если бы они там были)
- Обнаружить, что сервер принимает PHP-файлы (`1.php`), что может быть использовано для дальнейших атак

## Выводы о защищенности
В ходе анализа выявлена ошибка конфигурации: **directory listing не отключён** для директории `/hackable/uploads/`. Это позволяет любому пользователю просматривать список загруженных файлов и скачивать их.

В реальной атаке злоумышленник мог бы:
- Найти и скачать скомпилированные Java-классы из `WEB-INF/classes/` (если бы они там были)
- Декомпилировать их и проанализировать код на наличие уязвимостей (IDOR, SQL injection, хардкод паролей)
- Использовать найденные уязвимости для компрометации системы

- markdown
### Рекомендация по устранению
Для запрета directory listing необходимо в конфигурации Apache установить директиву:
`Options -Indexes`

Это отключит возможность просмотра списка файлов в директории `/hackable/uploads/` и предотвратит утечку информации.

Данная проблема классифицируется как:
- **CWE-200: Exposure of Sensitive Information** — раскрытие информации о загруженных файлах
- **CWE-548: Exposure of Information Through Directory Listing** — открытый листинг директории
